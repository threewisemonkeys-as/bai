<world_knowledge>
- When performing a directional movement action, the agent moves exactly one tile in that direction. Specifically:
    - `north` changes `(r, c)` to `(r-1, c)`.
    - `south` changes `(r, c)` to `(r+1, c)`.
    - `east` changes `(r, c)` to `(r, c+1)`.
    - `west` changes `(r, c)` to `(r, c-1)`.
    - `northeast` changes `(r, c)` to `(r-1, c+1)`.
    - `northwest` changes `(r, c)` to `(r-1, c-1)`.
    - `southeast` changes `(r, c)` to `(r+1, c+1)`.
    - `southwest` changes `(r, c)` to `(r+1, c-1)`.
- The `Movement Outcomes` provided by the perception module explicitly state the target coordinate and its associated status (Hazard, Monster (Combat), Wall/Obstacle, Safe, Unknown/Unseen) for *all possible single-step directional actions* from the agent's current position. This information is definitive for assessing movement safety and is the most reliable and up-to-date source of information for movement. A tile labeled "Safe" in `Movement Outcomes` indicates that movement *into* that tile is physically possible.
- The `search` action is not a universal solution for all blocked paths; it may not reveal hidden passages for certain barriers (e.g., continuous hazards like molten lava).
- The 'd' character on the map can represent an interactable door or a monster (e.g., jackal). If an `open` action on an adjacent 'd' yields "You see no door there," that specific 'd' location should be considered non-interactable as a door. If moving onto a 'd' tile results in combat, it signifies a monster. Some 'd' characters can also be aggressive, high-damage melee monsters that can interrupt player actions.
- The `down` action can only be successfully performed on stairs marked as `>`. If the message "You can't go down here." is received when attempting to descend `(` or `<` stairs, it means those are not the correct stairs to descend to the next level. `>` represents stairs down (the primary objective). `<` represents stairs up (not the objective for descending to the next level). There is currently no explicit action in the provided action set to ascend stairs marked `<`.
- **Zapping a 'wand of cold' (or an unidentified wand that is revealed to be a 'wand of cold') at molten lava (`}`) is a proven way to bypass this 'Hazard' by freezing and solidifying it into a traversable path.** Unidentified wands reveal their true nature (e.g., `wand of cold`) upon successful first use. If no specific method of bypass is known or available, especially for a continuous line of such "Hazards", the agent must seek alternative exploration paths that completely circumvent these barriers.
- If `apply` or `quaff` actions are attempted on a scroll and result in messages like "That is a silly thing to drink." or "That is a silly thing to use or apply.", it means these actions are not suitable for interacting with scrolls. Scrolls require a 'read' action which is not currently available in the agent's default action set. Therefore, scrolls cannot be directly used with the available actions.
- `zap`ping a wand can have directional effects. The `zap` action with a specified direction causes a magical bolt to travel *from* the agent *in* that direction. If the agent's body is in the path of the bolt, or if the nature of the spell allows, the agent can be harmed or killed by its own spell, especially with offensive wands (e.g., 'wand of cold'). Therefore, the agent must be cautious when zapping, ensuring the intended target is in the chosen direction and there's no risk of self-damage. The `zap` action can be interactive: executing `zap` without arguments often prompts for an item selection (e.g., `c` for 'wand of cold'), followed by a directional input (e.g., `east`). **When using an offensive wand (like 'wand of cold') for identification or utility without expecting self-damage, targeting an actual "Wall/Obstacle" or a truly distant, empty region is preferable over targeting an adjacent "Safe" tile.**
- If the perception module fails to provide complete or consistent critical information (e.g., `Movement Outcomes`, `Hazards`), assume uncertainty and prioritize defensive actions (e.g., `wait`, `search` locally for map updates, or `pray`) until reliable information is restored.
- **If any message follows an attempt to supply a follow-up input for a multi-step action (e.g., `eat`, `zap`), it indicates that the multi-step action was likely interrupted or completed with an unexpected outcome.** The agent must verify the effect of the action (e.g., check HP after `eat`, check status after `eat`) and re-initiate the original base command (e.g., `eat` again) to properly set up the context if the action needs to be repeated or completed.
- If `Items on Floor` are noted by ':', '?', or similar characters, they should be attempted to be acquired by moving onto their location. If moving to their location results in combat or a monster being killed, then the character at that location should be considered a monster, not just loot. Successful acquisition often occurs implicitly by moving onto the tile after killing a monster. If an explicit `pickup` action results in "There is nothing here to pick up" *immediately after killing a monster*, it indicates the item (its corpse) has already been collected implicitly.
- Multi-step actions (e.g., `eat`, `zap`, `apply`, `kick`, `wield`, `fire`) require multiple inputs. Once the first part of such an action is initiated, the game awaits further specific input (e.g., item selection, direction) before the turn fully ends and other entities (monsters) can act again. The agent's subsequent actions must prioritize completing the multi-step action's inputs.
- Eating when the agent's status is "Satiated" carries a risk of choking and death. The agent should avoid eating further if already Satiated.
- The status `Pw:X(Y)` refers to usable power or charges for wands/magical items, where `X` is the current charge and `Y` is the maximum charge. `zap` actions should consider available charges. Specific inventory items (e.g., `a wand of cold (1:5)`) also list their current and maximum charges.
- The `cursor: (x, y)` information present in the raw observation indicates the agent's absolute coordinates, interpreted as `(y, x)`. This data should be considered the ground truth for the agent's absolute position.
- Moving into a tile occupied by a `Monster` initiates combat and acts as an attack. This applies to `Monster` characters on the map, including 'd' (jackal/fox) and 'g' (goblin), as well as uppercase characters, which signal aggressive creatures that will pursue and attack the agent.
- A tile labeled "Safe" in `Movement Outcomes` can sometimes be occupied by a `Monster (Combat)`. Moving into such a "Safe" monster-occupied tile implies initiating combat. This reinforces the understanding that "Safe" implies traversable, with implied consequences, not devoid of all threat.
